How does a computer work?¶

The previous chapter was a “hook” in that it attempted to get the reader interested without really going into technical details. This chapter is a bit different. As is characteristic for this book, we won’t go very much into details but we will cover the essentials about understanding computers as is required for software development.

To reduce the modern computer to the essentials, let’s imagine a following, very simple device: a hand-held device with a touch screen. The device has very simple functionality: if the user taps on the left half of the screen, the screen turns green. If the user taps on the right half of the screen, the screen turns red. This device is very simple but will cover the same principles shared by nearly all computers, as we shall see.

In other words, there’s some input, some logic processing the input and creating some output. In our hypothetical device, the input is taps on the touch screen, and the output is what’s shown on the display. In our “Hello world” program, there was no input, and the output was a text in the terminal.

To be more specific, the input and output blocks like a touch screen and a display are digital devices; they communicate with the logic part using logic signals. A logic signal is an electric signal where its voltage is either 0 volts or some fixed, small voltage like 5 volts or 3.3 volts. Typically, instead of talking about voltages we instead talk about the value of a digital signal, such that e.g. 0 volts would mean 0, or false, and the higher voltage would mean 1, or true. (Logic signals are also called binary signals, whereby binary refers to a numeral system with only two symbols, 0 and 1, or binary digits, or bits.)

Generally speaking, the input and output devices come with some kind of a technical specification or data sheet which describe how to interact with them. For example, the data sheet for the touch screen could contain e.g. the following (somewhat simplified):

“The X coordinate of the latest tap is available at touch screen address 1. The value is between 0 (left border of the screen) and 100 (right border).”

Similarly the data sheet for the display could contain e.g.:

“To set the colour of the display, write the desired code to display address 1. The possible codes are: 0 = black; 1 = green; 2 = red; 3 = blue.”

(This display can only show one colour across the whole screen, i.e. it only has one pixel.)

The logic part in the middle ends up doing at least three things:

  • Receives information from the input blocks using logic signals
  • Performs the actual logic (in our case, mapping the position of the tap to the output colour)
  • Sends information to the output blocks using logic signals

This logic part is called the Central Processing Unit, or CPU. Its main responsibility is to execute the instructions (logic) provided to it, which typically causes communication with the input and output devices as a side effect.

In our case, the logic could look like this:

if user_tap_x_coordinate < 50:
    screen_color = 1
else:
    screen_color = 2

Here, the variable “user_tap_x_coordinate” represents the touch screen address 1 which holds the X coordinate of the position where the user tapped last. The variable “screen_color” represents the display address 1 which is used to control the colour shown on the display.

Then, “all” the CPU needs to do is read the value from the touch screen address, compare this with 50, and depending on the result of the comparison, write either 1 or 2 to the display address.

We now have some very rough understanding of how a computer works, but still haven’t really covered how a CPU works.

CPU¶

A CPU receives some code in some form and must execute it. Before we can understand how it performs this, we must understand digital logic.
